Tips & tricks

01

Claude Code for PoC

Of the LLM models I've tested, I think Claude Code is a powerful tool for building a PoC, both with and without a development background.

02

Keep focus on the core

Keep the interface limited to the most important business functions. Otherwise, the useful features disappear in a sea of less important ones. This is a challenge now that generated code can code up everything you ask for.

03

Reuse existing ML models

There are already many ML models that can be used. Check if they can be utilized before training your own. It can save both time and the environment.

04

Portability with Docker

AWS is excellent as infrastructure for all architectures, but using neutral software like Docker to create a layer between infrastructure and the system means you can quickly move to another cloud provider without rewriting applications.

05

Right cloud service for the right purpose requires exploration

Often there's much to gain by changing or adopting the right services for the right purpose. AWS has many resources to find the right architecture for your use case.

06

AI doesn't solve everything, but you should give it a chance

AI is not going to solve all efficiency or automation tasks, but if you don't give it a chance, someone else will, and then you lose competitiveness.
